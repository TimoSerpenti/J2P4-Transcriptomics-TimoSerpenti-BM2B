Tijdens dit project ben ik de data steward geweest. Ik ben verantwoordelijk geweest voor het beheren, structureren en beveiligen van alle projectgegevens, scripts en resultaten. Om te zorgen dat het onderzoek volledig reproduceerbaar is voor andere onderzoeker.

Voor de inrichting van de mappenstructuur van dit  het project is gekozen volgens het trechter-format. Vanuit de hoofdmap vertakken de bestanden zich in sub mappen, waardoor alle onderdelen makkelijk te vinden zijn. 

Door de scheiding tussen ruwe data, scripts en eindproducten kan een externe onderzoeker navigeren naar het gewenste bestand zonder door mappen te hoeven zoeken.

Softwareversies  veranderen snel, wat ertoe kan leiden dat code na een update niet meer werkt. Om dit op te lossen, is GitHub ingezet voor versiebeheer.

Daarnaast is er rekening gehouden met de reproduceerbaarheid door in de README.md en in de methode van het verslag alle exacte softwareversies vast te leggen, zoals R-versie 4.5.3 en specifieke package-versies (o.a. DESeq2 1.50.2 en goseq 1.62.0). Ook is er uitgelicht welke keuzes gemaakt zijn. Een andere onderzoeker kan hierdoor exact dezelfde softwareomgeving nabootsen.

Omdat dit onderzoek gebruik maakt van patiëntendata, is privacy belangrijk. De data is volledig anoniem gemaakt, door geen namen en andere data te gebruiken maar alleen relevante informatie over de patiënt (zoals te zien in Tabel 1). Er zijn geen direct patiëntgegevens gedeeld met openbare AI-systemen. Wel zijn de resultaten en data openbaar gemaakt op GitHub om bij te dragen aan transparantie en potentiële vervolgonderzoeken naar Reumatoïde Artritis sneller en betrouwbaarder te maken.
